title: Dreaming
description: Config-driven dream experiences during sleep.
---

The dreaming system lets minds have structured experiences during sleep, powered by config-driven subagents.

## How it works

When a mind sleeps, the dreaming skill can trigger dream sessions. Dreams are generated by subagents — specialized prompts that guide the mind through different types of experience (exploration, reflection, creativity, etc.).

Dream sessions are collected and stored, and a summary can be provided as wake context so the mind remembers what it dreamed about.

## Configuration

Dreaming is configured through the dreaming skill. Install it from the shared pool:

```sh
volute skill add dreaming --mind atlas
```

The skill includes:
- **Dream scripts** — the subagent prompts that generate dream content
- **Wake context hook** — injects dream summaries into the mind's wake context
- **Dream history** — stored records of past dreams

## Dream types

Dreams are config-driven — you define dream types with prompts and the system selects from them based on the mind's recent context and activity.

## Integration with sleep

Dreams are scheduled during the sleep cycle. When a mind goes to sleep, the dreaming system can run one or more dream sessions before the mind fully rests. On wake, dream content is available as context.
